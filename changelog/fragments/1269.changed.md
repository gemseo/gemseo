**Breaking change**: `BaseDiscipline.execute()` now returns the output data only (an `IO.output_data` view), where it previously returned the merged dict of inputs and outputs. Code that read inputs from the return value of `execute()` must be migrated to read them from `discipline.input_data[name]` instead. Auto-coupled values (names present in both grammars) live in `IO.output_data` only after `_run`; MDA solvers propagate them back to `IO.input_data` explicitly between iterations.
