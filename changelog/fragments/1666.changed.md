- `BaseFormulation` now takes an `EvaluationProblem` as input argument.
- The default value of the `name` argument of `Concatenate` is `""`; if so, its `name` attribute concatenates the names of the concatenated functions.
- BREAKING CHANGES for scenario:
    - `MDOScenario.save_optimization_history` was renamed to `MDOScenario.to_hdf` and `MDOScenario.to_ggobi`.
    - `MDOScenario.set_optimization_history_backup` was renamed to `MDOScenario.set_backup_settings`.
    - `MDOScenario.clear_history_before_execute` was renamed to `MDOScenario.clear_database_before_execute`.
    - `MDOScenario.to_dataset` exports all the samples in the case of a DOE algorithm, including replicates.
    - The output names must be passed as positional arguments in `EvaluationScenario.add_observable`, `MDOScenario.add_observable` and `MDOScenario.add_constraint`, e.g. `scenario.add_constraint("foo", "bar", value=1)` for outputs `"foo"` and `"bar"`.
    - The new signature of `MDOScenario` is `disciplines: Sequence[BaseDiscipline], design_space: DesignSpace, name: str = "", formulation_settings: BaseFormulationSettings | None = None)`.
    - The arguments `objective_name` and `maximize_objective` of `MDOScenario` have been removed to comply with the signature of the parent class `EvaluationScenario`; use the new method `add_objective` instead.
    - The arguments `objective_name` and `maximize_objective` of `OptAsMDOScenario` have been removed to comply with the signature of the parent class `MDOScenario`; use the new method `add_objective` instead.
    - `gemseo.scenarios.mdo_scenario` was renamed to `gemseo.scenarios.mdo`.
    - The single argument of `{EvaluationScenario,MDOScenario}.{execute,set_algorith}` is `algorithm_settings`.
    - The `Settings` attribute was removed because it was no longer useful.
